Feature Extraction

Extraction Methods

Each audio clip is summarized as a single vector of 17 numeric features. Each measurement is averaged across the clip's time frames so one number represents the whole recording. The same extraction code runs in both the training notebook and the deployed inference service.

Fundamental Frequency
The perceived pitch of the voice, estimated using the YIN algorithm. Only values in the 50–500 Hz singing range are kept; anything outside that range is treated as an extraction error. The mean pitch across all voiced frames is used.
Spectral Centroid
The "center of gravity" of the sound's frequency content. A higher spectral centroid means the voice sounds brighter or sharper. Computed per frame and averaged across the clip.
Spectral Rolloff
The frequency below which 85% of the sound's energy falls. Higher values mean the voice has more high-frequency content. Closely tracks spectral centroid (r = 0.89), so both features reflect the same timbral brightness property.
HNR (Harmonic-to-Noise Ratio)
Measures how clean and tonal the voice sounds versus airy and noisy. Computed using Praat's cross-correlation algorithm (time step = 0.01 s, minimum pitch = 50 Hz). Values below −20 dB indicate a failed extraction rather than a genuine vocal measurement and are excluded before modeling.
MFCCs (13 coefficients)
Mel-Frequency Cepstral Coefficients capture the tone color and texture of the voice in a compact form inspired by how the human auditory system processes sound. Each of the 13 coefficients describes a different scale of timbral detail, from overall loudness (mfcc_0) to fine harmonic texture (mfcc_1–12). Each coefficient is averaged across frames. Adding MFCCs to the feature set drove accuracy from around 70% to 92.8%, confirming they capture the vocal differences that simpler measurements miss.
Feature Scaling

Before training, all features are normalized using z-score standardization (StandardScaler). This is necessary because features have very different units and ranges: pitch is in Hz (50–500), HNR is in dB, and MFCC values are unitless. Without normalization, large-scale features would dominate the linear model. The scaler is embedded inside a scikit-learn Pipeline so normalization is learned only from training data and applied consistently at inference time.

Logistic Regression

Logistic Regression predicts class membership as a weighted combination of the input features, making it the most transparent of the three models. Each feature's coefficient directly shows its direction and strength of influence on the prediction. Adding MFCCs pushed accuracy from 58.6% to 91.4%. Although XGBoost reached 92.8%, Logistic Regression was selected for deployment because it offers nearly the same accuracy with simpler predictions, clear interpretability, and more reliable confidence scores on this small, well-structured feature set.

Random Forest

An ensemble of 100 decision trees, each trained on a random subset of the features and samples. This diversity makes Random Forest robust to outliers and non-linear patterns, such as the wide spread of HNR values in the data. It was also evaluated with 5-fold cross-validation to confirm generalization. Its Gini-based feature importance scores reveal which features contribute most to classification, providing interpretable insight into why MFCCs matter.

XGBoost

A gradient-boosted tree model that builds predictions iteratively. Each new tree focuses on correcting the mistakes made by the previous ones, gradually improving accuracy. XGBoost consistently performs well on structured tabular data and was included to test whether this sequential error-correction approach could outperform the parallel ensemble of Random Forest on this feature set. It achieved the highest overall accuracy at 92.8%.

Performance Metrics

Accuracy

The percentage of test samples classified correctly. This is the primary metric reported across all model variants. Because the dataset is nearly balanced between the two classes, accuracy gives a fair picture of model performance.

Precision & Recall

Per-class diagnostics. Precision measures how often a positive prediction is actually correct. Recall measures how many of the true positives the model successfully identified. Both are reported for each class via classification_report.

5-Fold Cross-Validation

Applied to Random Forest on the full dataset. The data is split into five equal folds; the model trains on four and tests on the remaining one, rotating through all folds. The mean accuracy across folds confirms whether the test-set result is stable or a statistical fluke.

ROC-AUC

The area under the ROC curve measures how well the model separates the two classes across all possible decision thresholds, not just the default 50% cutoff. A score of 1.0 is perfect separation; 0.5 is no better than random. Plotted for both Logistic Regression and Random Forest.
