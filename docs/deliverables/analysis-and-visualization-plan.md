# Analysis and Visualization Plan

## Table 1: Analysis Methods

| Method Name | Input to the Algorithm | Output of the Algorithm | What are you trying to show using this method? (Hypothesis, goals, and/or implications) | A brief explanation of the method / How is this method appropriate for your dataset or analysis |
|---|---|---|---|---|
| **PCA (Principal Component Analysis)** | Four acoustic features (f₀, Spectral Centroid, Spectral Rolloff, HNR) from ~2,550 VocalSet recordings across 10 vocal techniques. Features are z-score normalized so that differences in scale (Hz vs. dB) do not bias the components. | Two principal components (PC1, PC2) capturing the most variance; explained variance ratios per component; feature loadings showing each original feature's contribution. | **Goal:** Resolve the overlap between Spectral Centroid and Spectral Rolloff, which our EDA showed move together 89% of the time (r = 0.89), by combining the 4 features into 2 new summary features. **Hypothesis:** The 2D projection will show visible clustering between high-energy techniques (belt, forte, fast_forte, slow_forte, vibrato) and low-energy techniques (breathy, pp, slow_piano, vocal_fry, spoken). | PCA computes new axes (principal components) that maximize variance while being independent of each other. It fits our data because two of our four features — Spectral Centroid and Spectral Rolloff — move together 89% of the time, meaning they essentially measure the same thing. Keeping both would confuse linear models since they can't tell which one actually matters. PCA collapses them into independent components, and also lets us visualize whether the 10 techniques cluster by energy group in 2D. |
| **Logistic Regression** | Standardized features (or PCA components); binary labels (high_energy = 1, low_energy = 0) from grouping 10 techniques into two arousal categories. 80/20 stratified train-test split. | Predicted class labels and probability estimates; model coefficients showing each feature's direction and strength of influence. | **Goal:** Set a linear baseline for arousal classification accuracy. **Hypothesis:** A straight-line decision boundary will not fully separate the technique groups, since techniques like vibrato and vocal_fry may sit in overlapping acoustic space. This baseline quantifies how much non-linear models improve over it. | Logistic Regression predicts class membership using a weighted combination of input features. It works well as a baseline because the coefficients are directly interpretable — we can see exactly how much each acoustic feature pushes a prediction toward high or low energy. Its accuracy ceiling also tells us whether the problem needs non-linear models. |
| **Random Forest Classifier** | Standardized features (f₀, Spectral Centroid, Spectral Rolloff, HNR); binary labels (high_energy / low_energy). Evaluated with stratified 5-fold cross-validation. | Predicted class labels; Gini-based feature importance scores ranking the four features; out-of-bag error estimate for generalization. | **Goal:** Improve accuracy over the linear baseline and rank which features matter most. **Hypothesis:** HNR will be the most important feature even with 10 techniques, confirming that harmonic clarity vs. noise is the primary marker of vocal energy level. | Random Forest builds many independent decision trees on random subsets of the data and combines their votes. It fits our project because: (1) it handles non-linear patterns needed to separate techniques that overlap in some features; (2) it is robust to extreme values in our HNR data, which ranges from −86 to +24 dB due to extraction errors on some audio files; (3) it directly outputs feature importance scores, answering which feature best separates vocal energy levels. |
| **XGBoost (Gradient Boosting)** | Standardized features; binary labels. Hyperparameters (learning rate, max depth, number of trees) tuned via cross-validation. | Predicted class labels and probabilities; SHAP values showing each feature's contribution to every individual prediction. | **Goal:** Achieve the highest accuracy through sequential error correction, with SHAP for per-sample interpretability. **Hypothesis:** SHAP values will show that HNR drives predictions for breathy/pp/vocal_fry samples, while spectral centroid may matter more for distinguishing forte/belt — revealing that different techniques rely on different acoustic markers. | XGBoost builds decision trees one after another, where each new tree focuses on correcting the errors of the previous ones. It fits our tabular dataset (~2,550 samples, 4 features) because gradient boosting is consistently the top performer on structured data. SHAP analysis adds interpretability by showing whether different high-energy techniques (belt vs. forte vs. vibrato) depend on different features. |
| **RAVDESS Cross-Domain Validation** | Same four acoustic features extracted from RAVDESS sung emotion recordings (24 actors, 5 emotions: calm, happy, sad, angry, fearful). No retraining — the VocalSet-trained model is applied directly. | Predicted arousal labels for each RAVDESS recording; accuracy measured by mapping emotions to expected arousal (angry/happy → high, calm/sad → low). | **Goal:** Prove that vocal technique predicts emotional arousal. **Hypothesis:** A model trained only on technique labels will correctly classify emotional arousal — angry and happy singing predicted as high-energy, calm and sad as low-energy — without ever seeing emotion labels. This validates our core thesis. | RAVDESS contains labeled emotional singing from professional actors, providing an independent test of whether features that separate techniques also separate emotions. If a technique-trained model predicts emotion correctly, it proves that the acoustic markers of technique (especially HNR) are also markers of emotion. |
| **Spleeter Vocal Isolation + FMA Validation** | Full-mix audio tracks from the Free Music Archive (FMA), containing vocals mixed with instruments. | Isolated vocal stems from Spleeter's pre-trained separation model; then the same four features extracted from the vocals; predicted arousal labels from our trained model. | **Goal:** Test whether the classifier works on real-world music where vocals are AI-separated from a full mix. **Hypothesis:** Spleeter's separation artifacts will introduce noise that lowers HNR values (mimicking breathiness), so we expect accuracy to drop compared to VocalSet and RAVDESS. Measuring this drop shows how reliable HNR is on real-world audio. | Spleeter (by Deezer) is a pre-trained neural network that separates vocals from instruments. We need it because without isolation, instrumental energy contaminates our spectral features. FMA provides real music with genre metadata for indirect validation (e.g., metal → high-energy, ambient → low-energy). This is the final test of the full pipeline: raw song → vocal isolation → feature extraction → mood prediction. |
| **Music2Emo Valence Augmentation** | Same FMA audio tracks (full mix, not isolated vocals). | Valence score (1–9) and arousal score (1–9) per track from Music2Emo's pre-trained model. | **Goal:** Add valence (positive vs. negative emotion) to our arousal classification, achieving full Russell's Circumplex coverage. **Hypothesis:** Combining our vocal-based arousal with Music2Emo's valence will enable 4-quadrant mood tagging (happy, angry, sad, calm). Comparing Music2Emo's arousal against ours also tests whether our lightweight 4-feature approach agrees with a deep learning model. | Music2Emo is a pre-trained model that predicts both valence and arousal from audio. We use it specifically for valence, which vocal technique alone cannot capture — belt singing can be either happy or angry. By pairing our interpretable, vocal-based arousal with Music2Emo's valence, we get full circumplex coverage while keeping the arousal dimension explainable through vocal science. |

## Table 2: Visualization Plan

| Method Name | x-axis | y-axis | What are you trying to show, using this plot? |
|---|---|---|---|
| **PCA 2D Scatter Plot** | Principal Component 1 (PC1) | Principal Component 2 (PC2); points colored by energy class (blue = high-energy, orange = low-energy) | Show whether the 10 vocal techniques form two separable clusters by energy level in reduced feature space. This reveals if diverse techniques within each group (e.g., belt and vibrato are both high-energy) share similar acoustic properties. |
| **ROC Curves (Multi-Model Overlay)** | False Positive Rate (1 − Specificity) | True Positive Rate (Sensitivity); one curve per model (Logistic Regression, Random Forest, XGBoost), with AUC in legend | Compare how well each classifier separates high-energy from low-energy techniques across all thresholds. The model with the highest AUC has the best overall discrimination. |
| **Confusion Matrices (Per Model)** | Predicted class (high_energy / low_energy) | Actual class (high_energy / low_energy); cells show sample counts | Show each model's error profile — which energy class is harder to classify, and whether borderline techniques (e.g., vibrato, vocal_fry) cause most of the misclassifications. |
| **Feature Importance Bar Chart** | Feature name (f₀, Spectral Centroid, Spectral Rolloff, HNR) | Importance score: Gini importance (Random Forest) and SHAP mean |value| (XGBoost), side-by-side | Test whether HNR stays the dominant feature with all 10 techniques, or whether the expanded dataset shifts importance toward spectral features. Comparing across two models checks if the ranking is consistent. |
| **Three-Stage Accuracy Comparison** | Validation stage (VocalSet Test, RAVDESS Emotions, FMA Real-World) | Classification accuracy (%) for the best model | Show the generalization story: accuracy on lab data (VocalSet) → cross-domain emotion data (RAVDESS) → real-world music (FMA). Quantifies how much performance drops as conditions become more realistic. |
| **Russell's Circumplex Scatter Plot** | Valence score (Music2Emo, 1–9 scale) | Arousal prediction (our model, mapped to 1–9); each point is one FMA track, colored by genre | Map FMA predictions onto Russell's 2D emotion space: top-right = happy, top-left = angry, bottom-right = calm, bottom-left = sad. Validates whether genres land where expected (e.g., metal in the high-arousal region, ambient in low-arousal). |
